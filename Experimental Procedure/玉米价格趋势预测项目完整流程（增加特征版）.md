# 玉米未来30个交易日价格趋势预测项目流程（增加特征版）

目标：用过去60个交易日的玉米相关变量，预测未来30个交易日 dce_corn_close 是否明显上涨

# 一、项目总体思路

| 项目维度 | 统一设定 |
| --- | --- |
| 业务场景 | 可口可乐糖浆供应商采购玉米，用于生产果葡糖浆 |
| 目标价格 | dce_corn_close，即大商所玉米期货收盘价 |
| 预测任务 | 二分类趋势预测：未来30个交易日是否明显上涨 |
| 上涨阈值 | 未来30日收益率 > 0.5% （阈值）记为上涨，否则记为未明显上涨 |
| 主实验窗口 | 回看60个交易日，预测未来30个交易日 |
| 数据划分 | 按时间顺序8:1:1划分训练集、验证集和测试集 |
| 核心指标 | F1-score、AUC、Precision、Recall、Accuracy |

# 第一步：明确业务问题

- 把任务定义为采购决策辅助，而不是单纯的价格点位预测。

- 模型回答的问题是：未来30个交易日玉米价格会不会明显上涨。

- 如果模型预测上涨，供应商可以考虑提前采购、锁价或提高安全库存；如果预测未明显上涨，可以考虑分批采购或正常采购。

# 第二步：确定目标变量

- 以 dce_corn_close 作为核心目标价格。

- 计算未来30个交易日收益率：第 t+30 天价格除以第 t 天价格，再减去1。

- 当未来30日收益率大于0.5%时，标签记为1，表示明显上涨；否则记为0，表示未明显上涨。

- 最终目标变量命名为 target_up_30d_threshold。

- future_30d_return_for_label 只能用于生成标签，不能作为模型输入。（未来函数）

# 第四步：构造趋势特征

- 构造历史收益率，如10日、20日、30日、60日收益率。

- 构造移动均线，如10日、20日、30日、60日均线。

- 构造上涨持续性特征，例如过去10日、20日、30日上涨天数占比。

- 构造 Z-score 价格位置，用来判断当前价格处于历史高位、低位还是正常区间。

- 构造波动率特征，用来反映市场是否处于高波动状态。

- 构造基差特征，包括基差水平、基差率、基差均线、基差 Z-score 和基差相对均线偏离。

- 构造网络舆情和新闻特征，用来反映市场参与热度和资金情绪。

# 第五步：删除未来信息

- 生成标签后，最后30行没有未来30日价格，无法生成真实标签，需要删除。

- 建模前必须删除 date、future_30d_return_for_label、target_up_30d_threshold 和 split。

- 尤其不能把 future_30d_return_for_label 放进模型，因为它就是未来收益率，会造成未来函数。

# 第十二步：调整分类阈值

- 可以在验证集上尝试0.05、0.1等阈值。

- 最终阈值只能根据验证集确定，不能根据测试集反复调整。
