玉米价格二分类趋势预测实验标准口径

# 1. 项目目标

- 业务背景：糖浆供应商采购玉米用于制作果葡糖浆，需要预测未来玉米价格趋势以辅助进货决策。

- 主任务：预测未来 h 天玉米价格相对当前价格的上涨或下跌。

- 当前统一口径：趋势二分类。价格回归可作为辅助任务，不作为主任务。

# 2. 数据口径标准

| 标准项 | 统一口径 |
| --- | --- |
| 数据文件 | 使用 C:/时序玉米/玉米价格原始数据.csv。 |
| 目标价格 | 主价格序列保持一致；当前项目脚本使用 dce_corn_close。 |
| 时间顺序 | 按日期升序排列，时序任务禁止随机打乱。 |
| 缺失值 | 需要统一缺失值处理规则；训练、验证、测试保持同一处理方式。 |
| 异常值 | 如做截尾或修正，必须在实验记录中说明，且所有模型一致。 |
| 节假日 | 按交易日序列建模；如补齐自然日，必须全组统一。 |

# 3. 二分类趋势标签标准

| 标签项 | 定义 |
| --- | --- |
| 上涨 | y = 1，future_return_h > 0。 |
| 下跌或持平 | y = 0，future_return_h <= 0。 |
| 阈值 | 默认用 0 作为涨跌分界，不使用 0.5%/1% 震荡阈值。 |

future_return_h = (price[t+h] - price[t]) / price[t]
y_trend_h = 1 if future_return_h > 0 else 0

# 4. 回看步长与预测步长标准

| 维度 | 第一版统一取值 | 说明 |
| --- | --- | --- |
| 回看步长 lookback | 30,60 | 覆盖短期波动、半年周期、年度周期；长序列模型可加 730。 |
| 预测步长 horizon | 30,60,90 | 覆盖次日、一周、两周、一月和季度采购判断。 |

# 5. 训练/验证/测试切分标准

- 通用建议口径：训练集 80%，验证集 10%，测试集 10%。

# 6. 评价指标标准

| 指标 | 含义 | 使用定位 |
| --- | --- | --- |
| Accuracy | 整体预测正确比例。 | 辅助展示；类别不均衡时不能单独使用。 |
| Precision Up | 预测上涨的样本里，实际上涨的比例。 | 衡量提前采购建议是否可靠。 |
| AUC | 模型区分上涨/下跌的概率排序能力。 | 二分类主指标之一。 |
| Directional Accuracy | 涨跌方向预测正确比例。 | 若模型直接输出分类，基本等同 Accuracy；若输出价格/收益率，则很重要。 |
| MSE |  |  |
| MAE | 平均绝对误差，最好解释，比如平均差了多少元/吨 |  |
| R2 |  |  |

# 8. 模型输入输出标准

- 输入：过去 lookback 天的特征序列。

- 输出：未来 h 天上涨概率。

- 损失函数：二分类任务使用 Binary Cross Entropy。

- 分类阈值：默认 prob >= 0.5 判为上涨；如调阈值，只能在验证集上选择。

- 主线建议直接做分类；回归预测价格后再取正负可以作为单独实验线。

# 10. 采购业务模拟标准

- 预测上涨：提前采购。

- 预测下跌：延后采购。
